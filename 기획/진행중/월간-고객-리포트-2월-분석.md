## 5. AI 생성 리포트 품질 문제점

### 현재 AI 출력의 문제
1. **3주차/4주차 누락** — weekly_detail에 1~2주차만 포함 (max_tokens 부족 가능성)
2. **시스템 노이즈 미필터링** — "템플릿 초기화", "온보딩 스레드 수집", "활동 메모 포맷 정규화" 등이 그대로 출력
3. **비정형 백필 미구분** — W3의 258건이 실제 3주차 활동처럼 표시 (실제로는 W1~W2 소급 수집)
4. **Slack 댓글 잘림** — `[2026.02.11. 11:37 Slack 채널 댓글] 메일 확인...` 형태로 내용 truncate
5. **고객명 중복 미처리** — "코빗"과 "주식회사 코빗"이 별도 고객으로 집계
6. **overview에 코드블록 포함** — 마크다운 테이블이 ``` 안에 감싸져 출력
7. **비즈니스 인사이트 부재** — 수치 나열만 있고 "so what?" 해석 없음

## 6. 개선점 도출 — 월간 리포트 시스템 (`weeklyReportService.ts`)

### A. 데이터 전처리 개선 (DB → 프롬프트 전달 전)

| # | 문제 | 개선 방안 | 구현 위치 |
|---|------|---------|----------|
| A1 | 시스템 노이즈가 AI에 전달됨 | `change_type` 필터링 — `create`(템플릿 초기화), summary_title에 "온보딩 스레드 수집", "포맷 정규화" 포함 시 제외 | `collectMonthlyContextData()` |
| A2 | 비정형 백필 데이터가 실제 주차로 잘못 배정 | `change_type='batch_import'`인 항목은 `summary_title`에서 실제 날짜(`[비정형 채널 백필 2026.02.12...]`)를 파싱하여 해당 주차에 배정 | `collectMonthlyContextData()` |
| A3 | 고객명 중복 | DB 레벨에서 `company_name` 정규화 (alias 테이블 or CASE WHEN) — "주식회사 코빗"→"코빗", "비씨카드"→"BC카드" | 신규 테이블 or SQL |
| A4 | 개인명이 고객으로 집계 | `customer_contexts`에 `entity_type` (company/individual) 필드 추가, 리포트에서 개인은 별도 섹션으로 분리 | 스키마 변경 |
| A5 | Slack 댓글 내용 잘림 | `summary_title`에 `[Slack 채널 댓글]` 뒤 내용이 truncate됨 — 수집 시 요약을 더 길게 저장하거나, 원본 `input_text`에서 핵심 추출 | `CustomerContextManager` |

### B. 프롬프트 개선 (AI 호출)

| # | 문제 | 개선 방안 |
|---|------|---------|
