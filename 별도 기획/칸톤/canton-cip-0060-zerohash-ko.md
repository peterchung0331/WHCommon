## 개요 (Abstract)

- Zero Hash를 **최대 Weight 7.5**의 슈퍼 밸리데이터로 추가한다.
- Weight는 제품 통합, 교육/마케팅, 독점성 약속, 채택 인센티브 4개 영역의 마일스톤 달성에 따라 단계적으로 부여된다.
- Zero Hash의 약속 사항:
  - Canton Coin과 Canton 발행 USDC를 자사 제품 스위트에 통합
  - Canton을 프라이버시 기반 온체인 결제의 **독점 블록체인**으로 채택
  - 교육, 소셜미디어, 이벤트를 통해 Canton 생태계를 홍보
  - 자사 전략적 고객(Stripe, Shift4, Nuvei 등)을 Canton에 온보딩
- 관련 보상은 마일스톤 충족 및 토크노믹스 그룹 검증 전까지 **발행(mint)되지 않는다.**

## 마일스톤 및 Weight 획득 조건 (Deliverables)

총 4개의 마일스톤 영역으로 구성되며, **제품 통합 → 교육/마케팅 → 독점성 → 채택 인센티브** 순서로 진행된다.

### D1. Zero Hash 제품 통합 (+2.5)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Canton Coin 및 Canton 발행 USDC를 Zero Hash 제품 스위트에 통합. 기타 Canton 기반 자산도 Zero Hash 재량으로 포함 가능 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | **+2.5** |

### D2. 교육, 소셜, 이벤트 (+0.5)

| 항목 | 내용 |
|------|------|
| **달성 조건** | 기한 내 아래 각 1건 이상 실행: ① 웨비나(Webinar) ② 백서(White Paper) ③ 소셜미디어 홍보(Social Media Amplification) ④ 공동 이벤트(Joint Event) |
| **기한** | CIP 승인 후 **90일** |
| **획득 Weight** | **+0.5** |

### D3. Canton 독점성 (+0.5)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Zero Hash가 **프라이버시 기반 온체인 결제**에 대해 Canton을 **독점 블록체인**으로 사용. 기술적으로 가능한 모든 경우에 Canton을 통해 결제 처리 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | **+0.5** |
| **비고** | "기술적으로 가능한(technically possible)" 조건 — Canton에서 아직 지원되지 않는 자산의 경우 예외 허용 |

### D4. 채택 인센티브 (최대 +4.0)

두 가지 카테고리로 나뉘며, 제품 통합 완료 후 **365일** 동안 달성 가능하다.

#### 카테고리 1: 거래량 기반 인센티브

| 항목 | 내용 |
|------|------|
| **달성 조건** | Zero Hash가 Canton을 통해 처리하는 트래픽 거래량에 비례 |
| **획득 Weight** | **$1.5억(150M)** 거래량당 **+1** |

#### 카테고리 2: 전략적 채택자 인센티브

| 항목 | 내용 |
|------|------|
| **달성 조건** | Zero Hash가 아래 전략적 참여자를 Canton에 통합: ① 리테일 지갑 제공사 ② 결제 제공사 (예: Stripe, Bolt, Shift4, Nuvei) ③ 급여 제공사 (예: Remote.com) ④ 펀드 사용 사례 (예: Kalshi) ⑤ 기타 (TWG와 협의) |
| **획득 Weight** | 승인된 참여자당 **+1** |

> 카테고리 1과 2를 합산하여 **최대 +4.0** (전체 Max Weight 7.5에서 D1~D3의 3.5를 뺀 나머지)

#### 채택 인센티브 시기별 구조

| 기간 | 가용 Weight | 획득 기준 |
|------|-----------|----------|
| **1~6개월** (통합 후) | 최대 **1.5** | 고유 고객 1곳이 Canton에서 **$1.25억(125M)+** 결제 시 **+0.25**, 최대 1.5 |
| **7~12개월** (통합 후) | 최대 **1.0** | Canton에서 **$5억(500M)** 결제량당 **+0.5**, 최대 1.0 |
| **13~24개월** (통합 후) | 최대 **1.5** | (구체적 기준은 TWG와 협의) |

### Weight 합산 구조

```
D1: 제품 통합         = 2.5
D2: 교육/소셜/이벤트  = 0.5
D3: Canton 독점성     = 0.5
D4: 채택 인센티브     = 최대 4.0
─────────────────────────────
총합:                  최대 7.5
```

## 마일스톤 운영 메커니즘 (Milestone Mechanics)

- GSF 토크노믹스 위원회는 Zero Hash가 제안하는 각 참여자를 심의하고, 해당 참여자가 채택 인센티브 대상에 해당하는지 **명확히 통보**한다.
- Zero Hash는 **60일마다** GSF 토크노믹스 위원회와 미팅을 갖고 마일스톤 진행 보고서를 제출한다.

## SV 보상 메커니즘 (SV Reward Mechanics)

### 에스크로 SV 설정

1. GSF가 **`extraBeneficiary` PartyID**를 생성하여 에스크로 SV를 설정한다. 이때 SV Weight는 최대 획득 가능치(7.5)로 설정된다.
2. 신청자(Zero Hash)는 에스크로 설정 과정을 GSF 및 SV 노드 운영자와 직접 조율해야 한다.
3. 에스크로 SV 운영 비용은 **신청자가 전액 부담**한다.
4. 에스크로 SV는 블록별 보상을 발행(mint)하지 않는다 — 모든 보상은 **미청구 보상 풀(Unclaimed Rewards)**로 적립된다.
5. 기존 SV 운영자의 **2/3**가 에스크로 SV 노드의 전체 Weight 호스팅을 허용하도록 설정을 업데이트한다.

### 마일스톤 달성 시

1. 신청자가 **토크노믹스 워킹그룹(TWG)**에 마일스톤 완료 증빙을 제출한다.
2. 신청자가 해당 마일스톤에 대해 받아야 할 **Canton Coin 수량 계산서**를 제시한다.
3. TWG가 마일스톤 충족을 인정하고 계산에 동의하면:
   - **Tokenomics-Announce 메일링리스트**에 공지가 발송된다.
   - GSF가 `extraBeneficiary`를 해당 SV가 통제하는 **활성 PartyID**로 변경한다.
   - SV 운영자의 **2/3**가 미청구 보상 중 일부를 신청자의 Validator에 배정한다.

### 마일스톤 미달성 시

1. GSF가 신청자에게 미달성을 통보한다.
2. `extraBeneficiary` SV에 배정된 **남은 Weight가 GSF 노드에서 제거**되고, GSF SV 노드의 총 Weight가 같은 만큼 감소한다 (SV 투표로 결정).
3. TWG가 미청구 보상 처리 방안을 SV들에 **권고**한다.

### CIP-0045 적용

- 신청자는 **CIP-0045 (SV 운영 요건)**의 적용을 받는다.
- Weight가 **2.5를 초과**하는 시점부터 **6개월 이내에 자체 SV 노드를 운영**해야 한다.
- 자체 SV 노드는 Weight 0으로 네트워크에 합류하며, 마일스톤 달성에 따라 Weight가 증가한다.

## 분석 노트 — 웨이브릿지 관점

### Zero Hash의 차별점

1. **API 인프라 플랫폼**: Zodia나 Hex Trust 같은 커스터디 업체가 아닌, 결제/정산 **인프라 제공업체**. 고객사(Stripe, Bolt 등)가 암호화폐 결제를 자사 서비스에 내장할 수 있게 하는 백엔드
2. **독점성 조항**: Canton을 프라이버시 결제의 독점 블록체인으로 사용하겠다는 약속은 다른 CIP에서 보기 드문 패턴. 강력한 커밋먼트 표현
3. **교육/마케팅 의무**: 단순 기술 통합 외에 교육·마케팅 활동을 별도 마일스톤(+0.5)으로 분리. Canton 인지도 확산에 기여
4. **전략적 채택자 인센티브**: Stripe, Shift4 등 대형 결제 플랫폼을 Canton에 연결하는 것이 핵심 가치. 참여자당 +1 Weight로 높은 인센티브 부여

### 마일스톤 패턴 특징

- **통합(2.5) + 교육(0.5) + 독점성(0.5) + 채택(4.0)** = 4개 영역 분리
- 채택 인센티브의 **시기별 분할** (1~6개월 / 7~12개월 / 13~24개월)은 장기적 성장을 유도하는 구조
- 채택 기한이 **365일**(1년)로 타 CIP(180일)보다 길며, 거래량 + 참여자 수 **복합 기준** 적용

### 웨이브릿지 마일스톤 설계 시 참고

- **교육/마케팅 마일스톤** 분리 패턴 → 한국 금융기관 대상 Canton 교육 프로그램에 활용 가능
- **독점성 조항** → 한국 시장에서 Canton을 독점적 프라이버시 정산 인프라로 포지셔닝하는 약속으로 활용 가능
- **전략적 채택자 인센티브** 구조 → 한국 금융기관 온보딩을 기관당 Weight로 산정하는 구조에 적합
