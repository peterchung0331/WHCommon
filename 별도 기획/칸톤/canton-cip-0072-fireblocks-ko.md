## 개요 (Abstract)

- Fireblocks를 **최대 Weight 5**의 슈퍼 밸리데이터로 추가한다.

---

## 신청자 소개 (About Applicant)

### Fireblocks란?

디지털 자산의 세계가 속도, 규모, 복잡성 면에서 가속화되면서 블록체인을 활용하는 새로운 방식이 열리고 있다. Fireblocks의 **플랫폼과 네트워크**는 기업이 디지털 자산을 가장 간편하고 안전하게 활용할 수 있는 방법을 제공하며, 세계 최대 금융기관, 은행, 글로벌 브랜드, Web3 기업들이 신뢰하고 있다.

### 주요 고객

| 고객 | 유형 |
|------|------|
| **BNY Mellon** | 미국 최대 수탁 은행 |
| **BNP Paribas** | 유럽 대형 은행 |
| **ANZ Bank** | 호주·뉴질랜드 대형 은행 |
| **Revolut** | 영국 디지털 뱅킹 |
| 기타 수천 개 기관 | — |

### 핵심 지표

| 지표 | 수치 |
|------|------|

| 누적 거래 규모 | **$10조(10 trillion)+** |
| 보안 지갑 수 | **3억(300M)+** |

## 마일스톤 및 Weight 획득 조건 (Deliverables)

총 3개의 마일스톤으로 구성되며, **월렛 지원(Integration)** + **가속 보너스(Acceleration)** + **채택 보너스(Adoption)** 구조이다.

### D1. Canton 토큰 표준 월렛 지원 (+1)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Fireblocks 고객이 Canton 토큰 표준을 준수하는 모든 자산을 **자체 커스터디(self-custody)**할 수 있도록 월렛 지원 구현 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | **+1** |

### D2. 가속 보너스 (최대 +2)

| 항목 | 내용 |
|------|------|
| **달성 조건** | D1을 빨리 완료할수록 보너스 Weight를 더 받음 |
| **추가 요건** | Fireblocks 고객 최소 **1곳**이 MainNet에서 거래를 완료해야 함 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | 60일 이내 완료 시 **+2** (만점), 180일에 완료 시 **+0** (보너스 없음), 그 사이는 **선형 비례** (예: 120일 → +1) |

### D3. 채택 보너스 (최대 +2)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Fireblocks 기존 고객의 Canton 채택을 촉진 |
| **기한** | 월렛 지원 Go Live 후 **180일** |
| **획득 Weight** | 이전된 자산 **$15억(1.5B)당 +0.5**, 최대 **+2** |

### Weight 합산 구조

```
월렛 지원(Integration):  +1
가속 보너스(Acceleration): 60일 +2 ~ 180일 +0 (선형 비례), 최대 +2
채택 보너스(Adoption):     $1.5B당 +0.5, 최대 +2
────────────────────────────────────────────────────────
총합:                                         최대 5.0
```

## SV 보상 메커니즘 (SV Reward Mechanics)

### 에스크로 SV 설정

1. GSF가 **`extraBeneficiary` PartyID**를 생성하여 에스크로 SV를 설정한다. 이때 SV Weight는 최대 획득 가능치(5)로 설정된다. "First to Production" 또는 "Second to Production" 보너스 Weight는 포함하지 않는다.
2. 신청자(Fireblocks)는 에스크로 설정 과정을 GSF 및 GSF SV 노드 운영자와 직접 조율해야 한다.
3. 에스크로 SV 운영 비용은 **신청자가 전액 부담**한다.
4. 에스크로 SV는 블록별 보상을 발행(mint)하지 않는다 — 모든 보상은 **미청구 보상 풀(Unclaimed Rewards)**로 적립된다.
5. 기존 SV 운영자의 **2/3**가 GSF가 해당 SV의 전체 Weight를 호스팅하도록 설정을 업데이트한다.

### 마일스톤 달성 시

1. 신청자가 **토크노믹스 워킹그룹(TWG)**에 마일스톤 완료 증빙을 제출한다.
2. 신청자가 해당 마일스톤에 대해 받아야 할 **Canton Coin 수량 계산서**를 제시한다.
3. TWG가 마일스톤 충족을 인정하고 계산에 동의하면:
   - **Tokenomics-Announce 메일링리스트**에 공지가 발송된다.
   - GSF가 `extraBeneficiary`를 해당 SV가 통제하는 **활성 PartyID**로 변경한다.
   - SV 운영자의 **2/3**가 미청구 보상 중 일부를 신청자의 Validator에 배정한다.

### 마일스톤 미달성 시

1. GSF가 신청자에게 미달성을 통보한다.
2. `extraBeneficiary` SV에 배정된 **남은 Weight가 GSF 노드에서 제거**되고, GSF SV 노드의 총 Weight가 같은 만큼 감소한다 (SV 투표로 결정).
3. TWG가 미청구 보상 처리 방안을 SV들에 **권고**한다.

### CIP-0045 적용

- 신청자는 **CIP-0045 (SV 운영 요건)**의 적용을 받는다.
- Weight가 **2.5를 초과**하는 시점부터 **6개월 이내에 자체 SV 노드를 운영**해야 한다.
- 자체 SV 노드는 Weight 0으로 네트워크에 합류하며, 마일스톤 달성에 따라 Weight가 증가한다.
