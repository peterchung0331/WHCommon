## 개요 (Abstract)

- BitGo를 **최대 Weight 5**의 슈퍼 밸리데이터로 추가한다.

---

## 신청자 소개 (About Applicant)

### BitGo란?

BitGo는 디지털 자산 솔루션의 **선도적 인프라 제공자**로, 규제 준수 콜드 스토리지에서 **커스터디, 월렛, 스테이킹, 트레이딩, 파이낸싱, 정산 서비스**를 제공한다.

2013년 설립 이후, 고객이 디지털 자산 공간을 안전하게 탐색할 수 있도록 하는 데 집중해왔다. 전 세계 다수의 규제 법인을 통한 글로벌 프레즌스를 갖추고 있으며, 업계 주요 브랜드, 거래소, 플랫폼을 포함한 수천 개 기관과 전 세계 수백만 리테일 투자자에게 서비스를 제공한다.

**디지털 경제의 운영 백본(Operational Backbone)**으로서, BitGo는 비트코인 네트워크 거래의 상당 부분을 처리하며, **세계 최대 독립 디지털 자산 커스터디안이자 스테이킹 제공자**이다.

### 핵심 지표

| 지표 | 수치 |
|------|------|
| 기관 고객 수 | **2,000+** (90+ 국가) |
| 글로벌 거래소 파트너 | **100+** |
| 지원 코인/토큰 | **1,200+** |

## 마일스톤 및 Weight 획득 조건 (Deliverables)

총 3개의 마일스톤으로 구성되며, **커스터디 지원(Integration)** + **가속 보너스(Acceleration)** + **채택 보너스(Adoption)** 구조이다.

> **참고**: Fireblocks(CIP-0072)와 거의 동일한 마일스톤 구조를 가진다. 차이점은 Fireblocks가 "월렛 지원(self-custody)"인 반면, BitGo는 "커스터디 지원(custody)"이라는 표현을 사용한다.

### D1. Canton 토큰 표준 커스터디 지원 (+1)

| 항목 | 내용 |
|------|------|
| **달성 조건** | BitGo 고객이 Canton 토큰 표준을 준수하는 모든 자산을 **커스터디(custody)**할 수 있도록 구현 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | **+1** |

### D2. 가속 보너스 (최대 +2)

| 항목 | 내용 |
|------|------|
| **달성 조건** | D1을 빨리 완료할수록 보너스 Weight를 더 받음 |
| **추가 요건** | BitGo 고객 최소 **1곳**이 MainNet에서 거래를 완료해야 함 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | 60일 이내 완료 시 **+2** (만점), 180일에 완료 시 **+0** (보너스 없음), 그 사이는 **선형 비례** (예: 120일 → +1) |

### D3. 채택 보너스 (최대 +2)

| 항목 | 내용 |
|------|------|
| **달성 조건** | BitGo 기존 고객의 Canton 채택을 촉진 |
| **기한** | 커스터디 지원 Go Live 후 **180일** |
| **획득 Weight** | 이전된 자산 **$15억(1.5B)당 +0.5**, 최대 **+2** |

### Weight 합산 구조

```
커스터디 지원(Integration):  +1
가속 보너스(Acceleration):   60일 +2 ~ 180일 +0 (선형 비례), 최대 +2
채택 보너스(Adoption):       $1.5B당 +0.5, 최대 +2
────────────────────────────────────────────────────────
총합:                                          최대 5.0
```

## SV 보상 메커니즘 (SV Reward Mechanics)

### 에스크로 SV 설정

1. GSF가 **`extraBeneficiary` PartyID**를 생성하여 에스크로 SV를 설정한다. 이때 SV Weight는 최대 획득 가능치(5)로 설정된다. "First to Production" 또는 "Second to Production" 보너스 Weight는 포함하지 않는다.
2. 신청자(BitGo)는 에스크로 설정 과정을 GSF 및 GSF SV 노드 운영자와 직접 조율해야 한다.
3. 에스크로 SV 운영 비용은 **신청자가 전액 부담**한다.
4. 에스크로 SV는 블록별 보상을 발행(mint)하지 않는다 — 모든 보상은 **미청구 보상 풀(Unclaimed Rewards)**로 적립된다.
5. 기존 SV 운영자의 **2/3**가 GSF가 해당 SV의 전체 Weight를 호스팅하도록 설정을 업데이트한다.

### 마일스톤 달성 시

1. 신청자가 **토크노믹스 워킹그룹(TWG)**에 마일스톤 완료 증빙을 제출한다.
2. 신청자가 해당 마일스톤에 대해 받아야 할 **Canton Coin 수량 계산서**를 제시한다.
3. TWG가 마일스톤 충족을 인정하고 계산에 동의하면:
   - **Tokenomics-Announce 메일링리스트**에 공지가 발송된다.
   - GSF가 `extraBeneficiary`를 해당 SV가 통제하는 **활성 PartyID**로 변경한다.
   - SV 운영자의 **2/3**가 미청구 보상 중 일부를 신청자의 Validator에 배정한다.

### 마일스톤 미달성 시

1. GSF가 신청자에게 미달성을 통보한다.
2. `extraBeneficiary` SV에 배정된 **남은 Weight가 GSF 노드에서 제거**되고, GSF SV 노드의 총 Weight가 같은 만큼 감소한다 (SV 투표로 결정).
3. TWG가 미청구 보상 처리 방안을 SV들에 **권고**한다.

### CIP-0045 적용

- 신청자는 **CIP-0045 (SV 운영 요건)**의 적용을 받는다.
- Weight가 **2.5를 초과**하는 시점부터 **6개월 이내에 자체 SV 노드를 운영**해야 한다.
- 자체 SV 노드는 Weight 0으로 네트워크에 합류하며, 마일스톤 달성에 따라 Weight가 증가한다.

## Fireblocks(CIP-0072)와의 비교

| 항목 | BitGo (CIP-0074) | Fireblocks (CIP-0072) |
|------|-------------------|----------------------|
| Max Weight | 5 | 5 |
| D1 명칭 | Custody Support | Wallet Support |
| D1 성격 | 커스터디(위탁 보관) | 셀프 커스터디(자체 보관) |
| D1 Weight | +1 | +1 |
| 가속 보너스 | 60일 +2 ~ 180일 +0 | 60일 +2 ~ 180일 +0 |
| 채택 보너스 단위 | $1.5B당 +0.5 | $1.5B당 +0.5 |
| 채택 보너스 최대 | +2 | +2 |
| 저자 | Eric Saraniecki, Jacob McCrum | Eric Saraniecki |
| 승인일 | 2025-08-27 | 2025-08-11 |

> BitGo와 Fireblocks는 사실상 **동일한 마일스톤 템플릿**을 사용한다. 핵심 차이는 BitGo가 "커스터디(custody)" — 즉 BitGo 플랫폼에서 고객 자산을 위탁 보관하는 방식인 반면, Fireblocks는 "셀프 커스터디(self-custody)" — 즉 Fireblocks 플랫폼을 통해 고객이 직접 자산을 관리하는 방식이라는 점이다.
