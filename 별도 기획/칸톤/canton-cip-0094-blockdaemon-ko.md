## 개요 (Abstract)

- Blockdaemon을 **최대 Weight 5**의 슈퍼 밸리데이터로 추가한다.
- Weight는 3개의 마일스톤 달성에 따라 **270일**에 걸쳐 단계적으로 부여된다.
- Blockdaemon의 약속 사항:
  - 지리적으로 분산된 멀티클라우드 밸리데이터 클러스터를 운영하며 **감사를 받은 99.9%+ 업타임**을 유지
  - 규제를 받는 금융기관이 Canton Network에 참여할 수 있도록 기관급 인프라를 제공
- 관련 보상은 마일스톤 충족 및 토크노믹스 그룹 검증 전까지 **발행(mint)되지 않는다.**

## 신청자 소개 (About Applicant)

### Blockdaemon이란?

Blockdaemon은 **블록체인 인프라 플랫폼 전문 기업**으로, 기관 고객을 위한 노드 운영, 스테이킹, 지갑 인프라, API 서비스를 제공한다.

### 주요 현황

| 항목 | 내용 |
|------|------|
| **관리 노드** | 40개+ 글로벌 거점에서 **25만 개** 노드 관리 |
| **기관 고객** | **400개+** 기관 |
| **보안 인증** | **ISO 27001** 및 **SOC 2 Type II** 인증 |
| **컴플라이언스** | OFAC 스크리닝 준수 |

### 포지셔닝

Blockdaemon은 **블록체인 노드 인프라의 AWS**와 같은 포지션을 목표로 한다. 금융기관이 블록체인 프로토콜에 참여할 때 직접 노드를 운영하는 대신 Blockdaemon의 관리형 인프라를 이용할 수 있으며, 이는 Canton Network의 **기관 채택 장벽**을 낮추는 데 직접 기여한다.

### 핵심 서비스

| 서비스 | 설명 |
|--------|------|
| **Managed Nodes** | 기관 대상 관리형 블록체인 노드 운영 |
| **Staking** | 기관급 스테이킹 인프라 및 보상 관리 |
| **Wallet Infrastructure** | 기관 지갑 생성·관리·보안 인프라 |
| **APIs & Tools** | 블록체인 데이터 접근 및 트랜잭션 처리 API |

## 마일스톤 및 Weight 획득 조건 (Deliverables)

총 3개의 마일스톤으로 구성되며, **지갑 통합 → 거래량 → TVL 채택** 순서로 진행된다. 전체 기간은 최대 **270일**.

### D1. 기관 지갑 지원 (+1.0)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Canton 통합 지갑 배포 — 인증 지갑(Authenticated Wallets), 외부 당사자 수명주기 관리(External Party Lifecycle Management), 네이티브 Canton 및 CIP-0056 토큰 표준 지원 포함. 첫 번째 고객이 **$1,000만(10M)+** 자산을 보유하는 상태에서 운영 |
| **기한** | CIP 승인 후 **180일** |
| **획득 Weight** | **+1.0** |

### D2. 거래량 보너스 (최대 +2.0)

| 항목 | 내용 |
|------|------|
| **달성 조건** | Blockdaemon 인프라를 통해 실행되는 기관 거래 건수에 비례 |
| **기한** | CIP 승인 후 **270일** |
| **획득 Weight** | **1,000만 건(10M)** 거래당 **+0.5**, 최대 **+2.0** |

> 즉, 4,000만 건(40M) 거래 달성 시 만점

### D3. 채택 보너스 (최대 +2.0)

| 항목 | 내용 |
|------|------|
| **달성 조건** | 신규 고객 또는 새로운 거래 유형을 통해 Canton에 유입되는 TVL(Total Value Locked)에 비례 |
| **기한** | CIP 승인 후 **270일** |
| **획득 Weight** | 신규 고객의 TVL **$5억(500M)** 추가당 **+0.5**, 최대 **+2.0** |

> 즉, $20억(2B) TVL 달성 시 만점

### Weight 합산 구조

```
D1: 기관 지갑 지원       = 1.0
D2: 거래량 보너스        = 최대 2.0
D3: 채택 보너스 (TVL)    = 최대 2.0
──────────────────────────────────
총합:                     최대 5.0
```

## SV 보상 메커니즘 (SV Reward Mechanics)

### 에스크로 SV 설정

1. GSF가 **`extraBeneficiary` PartyID**를 생성하여 에스크로 SV를 설정한다. 이때 SV Weight는 최대 획득 가능치(5)로 설정된다.
2. 신청자(Blockdaemon)는 에스크로 설정 과정을 GSF 및 SV 노드 운영자와 직접 조율해야 한다.
3. 에스크로 SV 운영 비용은 **신청자가 전액 부담**한다.
4. 에스크로 SV는 블록별 보상을 발행(mint)하지 않는다 — 모든 보상은 **미청구 보상 풀(Unclaimed Rewards)**로 적립된다.
5. 기존 SV 운영자의 **2/3**가 에스크로 SV 노드의 전체 Weight 호스팅을 허용하도록 설정을 업데이트한다.

### 마일스톤 달성 시

1. 신청자가 **토크노믹스 워킹그룹(TWG)**에 마일스톤 완료 증빙을 제출한다.
2. 신청자가 해당 마일스톤에 대해 받아야 할 **Canton Coin 수량 계산서**를 제시한다.
3. TWG가 마일스톤 충족을 인정하고 계산에 동의하면:
   - **Tokenomics-Announce 메일링리스트**에 공지가 발송된다.
   - GSF가 `extraBeneficiary`를 해당 SV가 통제하는 **활성 PartyID**로 변경한다.
   - SV 운영자의 **2/3**가 미청구 보상 중 일부를 신청자의 Validator에 배정한다.

### 마일스톤 미달성 시

1. GSF가 신청자에게 미달성을 통보한다.
2. `extraBeneficiary` SV에 배정된 **남은 Weight가 GSF 노드에서 제거**되고, GSF SV 노드의 총 Weight가 같은 만큼 감소한다 (SV 투표로 결정).
3. TWG가 미청구 보상 처리 방안을 SV들에 **권고**한다.

### CIP-0045 적용

- 신청자는 **CIP-0045 (SV 운영 요건)**의 적용을 받는다.
- Weight가 **2.5를 초과**하는 시점부터 **6개월 이내에 자체 SV 노드를 운영**해야 한다.
- 자체 SV 노드는 Weight 0으로 네트워크에 합류하며, 마일스톤 달성에 따라 Weight가 증가한다.

## 분석 노트 — 웨이브릿지 관점

### Blockdaemon의 차별점

1. **인프라 전문 업체**: 커스터디나 트레이딩이 아닌 **노드 운영 및 지갑 인프라** 전문. Canton Network의 기술적 접근성을 높이는 역할
2. **대규모 노드 관리 실적**: 25만 개 노드, 40개+ 거점, 400개+ 기관 — 블록체인 인프라 분야에서 가장 큰 규모
3. **보안 인증**: ISO 27001 + SOC 2 Type II 인증으로 기관 고객의 컴플라이언스 요건 충족
4. **정량적 마일스톤**: 거래량(건수)과 TVL이라는 매우 명확한 정량 지표 사용

### 마일스톤 패턴 특징

- **3개 마일스톤만으로 단순 구조**: 가속 보너스 없이 통합(1.0) + 거래량(2.0) + TVL(2.0)
- **거래 "건수" 기반** 거래량 측정 — 금액 기반이 아닌 건수 기반은 다른 CIP와 차별화
- **270일** 기한으로 타 CIP(180일)보다 여유 있는 일정
- **가속 보너스가 없음** — 조기 달성 인센티브 없이 순수 성과 연동

### 웨이브릿지 마일스톤 설계 시 참고

- **거래 건수 vs 거래 금액** 중 어떤 지표를 사용할지 검토 필요 — Blockdaemon은 건수, Zodia/Talos는 금액 기준
- **TVL 기반 채택 보너스** 구조는 커스터디/금융 서비스에 적합 — 웨이브릿지의 한국 기관 자산 유치에 활용 가능
- **단순 3단계 구조**는 TWG 심사에서 명확성이 높아 승인에 유리할 수 있음
