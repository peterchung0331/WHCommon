## 1. 개요

### 1.1 목적

신규 가상자산 거래지원 시 자금세탁방지(AML) 관점에서의 위험을 체계적으로 평가하고, 거래지원 가부를 판단하기 위한 표준 업무 절차를 정의한다.

### 1.2 적용 범위

- **대상**: 당사에서 신규로 거래지원(커스터디)하고자 하는 모든 가상자산
- **평가 유형**: 신규 거래지원, 재평가
- **평가 주체**: 주관부서(사업운영팀/플랫폼운영팀) → 전담부서(AML팀)

### 1.3 평가서 구성

평가서는 5개 섹션으로 구성된다:

| 섹션 | 내용 | 성격 |
|------|------|------|
| 1. 평가 정보 | 평가 수행 메타 정보 | 관리 정보 |
| 2. 가상자산 상세 정보 | 대상 자산 기본 정보 | 리서치 |
| 3. 거래지원 적격성 평가 | 거래 가부 선결 조건 (3개) | Gate 판단 |
| 4. 가상자산 위험평가 | 고유위험(10개 항목) + 통제효과성(3개) | 정량/정성 평가 |
| 5. 종합의견 | 주관부서 + 전담부서 의견 | 최종 판단 |

## 2. 평가 프로세스 흐름

### 2.1 현행 프로세스 (수동)

```
[거래지원 요청 접수]
    ↓
[담당자 배정]
    ↓
[리서치 수행] ← 담당자가 직접 웹 조사
    - 법인/단체 정보 조사
    - 국가 정보 확인
    - 대표자/핵심인력 조사
    - 부정적 이슈 검색
    - 거래소 상장 현황 확인
    - 블록체인 추적 가능성 확인
    ↓
[평가서 작성] ← 엑셀 양식에 수기 입력
    ↓
[주관부서 종합의견 작성]
    ↓
[전담부서(AML) 리뷰 및 종합의견 작성]
    ↓
[최종 결과 확정]
    ↓
[WLF 검색 첨부] ← 별도 시스템
```

### 2.2 자동화 후 프로세스 (목표)

```
[거래지원 요청 접수]
    ↓
[담당자 가상자산 티커/이름 입력]
    ↓
[Claude API 1차 리서치 초안 자동 생성]  ← AI 자동화
    - 기본 정보 자동 수집
    - 적격성 자동 판별
    - 고유위험 점수 자동 산출
    - 종합의견 초안 자동 생성
    ↓
[플랫폼 운영팀 1차 리뷰]  ← 수동 확인
    - AI 리서치 결과 검증
    - 수동 확인 항목 보완 (특수관계인, 서류 제출 등)
    - 주관부서 종합의견 확정
    ↓
[AML팀 리뷰 & 승인]  ← 최종 판단
    - 전담부서 종합의견 작성
    - 최종 결과 확정 (거래지원 가능/불가)
    ↓
[WLF 검색 첨부] ← 별도 시스템 (자동화 범위 외)
```

### 3.2 섹션 2: 가상자산 상세 정보

각 항목별 조사 방법을 상세히 기술한다.

#### 2-1) 가상자산 명

- **형식**: 한글명(영문명)
- **조사 소스**: CoinMarketCap, CoinGecko, 국내 거래소 공시
- **예시**: "아반티스(Avantis)", "리콜 (Recall)", "비앤비 (BNB)"

#### 2-2) 발행 법인 및 단체명

- **조사 방법**:
  1. 공식 홈페이지 → About/Team 페이지 확인
  2. 백서(Whitepaper) 내 법인 정보 확인
  3. 블록체인 익스플로러의 토큰 정보 페이지
  4. Crunchbase, LinkedIn 법인 검색
  5. 국내 거래소 공시 자료 (빗썸, 업비트 등)
- **기재 방법**: 법인명과 역할을 함께 기재
  - 예: "Avantis Foundation(운영), Avantis BVI Ltd(발행)"
  - 예: "Atlas Networks Limited, Recall Foundation"

#### 2-3) 발행 법인 및 단체 소재 국가

3개 하위 항목을 각각 조사한다:

| 하위 항목 | 설명 | 조사 방법 |
|----------|------|----------|
| 1) 국적 | 법인 설립 국가 | 법인 등기, 백서, 공식 사이트 |
| 2) 소재지 | 실제 운영 소재지 | 공식 사이트, LinkedIn 본사 위치 |
| 3) 관할권 | 법적 관할 국가 | 이용약관, 법적 고지 확인 |

- **주의사항**:
  - 3개가 동일한 경우가 대부분이나, 다를 수 있음 (예: 케이맨 설립, 싱가포르 운영)
  - 국가명은 국가위험점수표의 한글명과 일치시킬 것 (VLOOKUP 연동)
  - **FATF 고위험 국가에 해당하면 섹션 3의 위험국가 여부에 영향**

#### 2-4) 가상자산 네트워크 계열

- **조사 소스**: CoinGecko/CoinMarketCap → Platform/Chain 정보, 블록체인 익스플로러
- **예시**: Ethereum, Base, BNB Smart Chain, Solana 등
- **기재**: 메인넷 이름 기재 (예: "Base", "Ethereum network", "bsc")

#### 2-5) 가상자산 백서

- **조사 방법**:
  1. 공식 홈페이지의 Docs/Whitepaper 섹션
  2. 거래소 공시 자료에 첨부된 백서 링크
  3. GitHub 리포지토리
- **기재**: URL 직접 링크 (복수 가능, 쉼표 구분)
- **예시**: "https://docs.recall.network/, https://feed-content.bithumb.com/cms/..."

#### 2-6) 가상자산 특징 및 내용

- **작성 방법**: 3~5문장으로 가상자산의 핵심 기능과 목적을 요약
- **포함 내용**:
  - 프로젝트의 핵심 목적/기능
  - 기술적 차별점
  - 토큰의 용도 (거버넌스, 유틸리티, 수수료 등)
- **조사 소스**: 백서, 공식 홈페이지, 거래소 공시 자료
- **참고**: 기존 평가 사례의 작성 패턴을 따름

#### 2-7) 가상자산 이용 대상

- **선택지**: 개인/개인사업자, 법인/단체 (복수 선택 가능)
- **판단 기준**: 대부분의 가상자산은 양쪽 모두 해당
- **예외 사례**: B2B 전용 토큰 등은 법인/단체만 해당

#### 2-8) 가상자산 제공 채널

- **선택지**: 대면, 비대면 (복수 선택 가능)
- **판단 기준**: 가상자산 거래는 대부분 비대면
- **기재**: 거의 모든 경우 "비대면"만 선택

#### 2-9) 거래지원 중인 거래소

- **조사 방법**:
  1. CoinMarketCap → Markets 탭에서 국내 거래소 필터
  2. CoinGecko → Exchanges 탭
  3. 각 국내 거래소 직접 확인 (업비트, 빗썸, 코인원, 코빗, 고팍스)
- **기재 형식**: "거래소1, 거래소2 (조사일자 기준)"
- **예시**: "업비트, 빗썸, 코인원 (2026.2.3 기준)"
- **중요**: 이 정보는 섹션 4-1의 "국내 거래소 2곳+" 평가에 직접 사용됨

### 3.3 섹션 3: 거래지원 적격성 평가

적격성 평가는 **선결 조건**으로, 3개 항목 중 하나라도 YES이면 **거래지원 불가**로 즉시 판정된다.

#### 3-1-1) 다크코인 여부

| 구분 | 내용 |
|------|------|
| **질문** | 거래지원하고자 하는 가상자산이 다크코인 혹은 프라이버시 코인인가? |
| **판단 기준** | CoinMarketCap Privacy 카테고리에 포함 여부 |
| **YES 예시** | Monero(XMR), Zcash(ZEC), Dash, Secret(SCRT) |
| **NO 예시** | 일반 유틸리티/거버넌스 토큰 |
| **조사 소스** | CoinMarketCap Privacy 카테고리 페이지 |
| **결과** | YES → 거래지원 불가 / NO → 다음 항목 |

#### 3-1-2) 특수관계인 여부

| 구분 | 내용 |
|------|------|
| **질문** | 거래지원하고자 하는 가상자산을 발행하는 법인 및 단체가 당사와 특수관계인에 해당하는가? |
| **판단 기준** | 내부 특수관계인 목록 대조 |
| **조사 방법** | 내부 데이터 확인 (AI 자동화 불가, 수동 확인 필수) |
| **결과** | YES → 거래지원 불가 / NO → 다음 항목 |

#### 3-1-3) 위험국가 여부

| 구분 | 내용 |
|------|------|
| **질문** | 발행 법인의 국적, 소재지 또는 관할권이 FATF 조치를 요하는 고위험 국가에 해당하는가? |
| **판단 기준** | 섹션 2-3의 국적/소재지/관할권 중 하나라도 FATF 고위험 국가 리스트에 포함되면 YES |
| **FATF 고위험 국가** | [6. FATF 고위험 국가 목록](#6-fatf-고위험-국가-목록) 참조 |
| **주의** | 버진아일랜드는 FATF 관찰대상국에 해당하나, 적격성 평가에서 NO로 판단된 사례 있음 (별도 논의 필요) |
| **결과** | YES → 거래지원 불가 / NO → 적격성 통과 |

#### 적격성 판정 수식

```
적격성 결과 = IF(다크코인=YES OR 특수관계인=YES OR 위험국가=YES, "거래지원 불가", "거래지원 가능")
```

> **참고**: 적격성 평가 통과 시에만 섹션 4 위험평가로 진행한다.

### 3.4 섹션 4-1: 고유위험 평가

고유위험 평가는 **10개 채점 항목**으로 구성되며, 각 항목별로 0~10점 범위에서 채점된다. 총점 100점 만점이며, **점수가 높을수록 위험도가 높다**.

#### 항목 1) 법인 및 단체의 정보 (택1, 3.33~10점)

법인 실체를 확인할 수 있는 서류의 제출 수준에 따라 채점한다.

| 선택지 | 점수 | 설명 |
|--------|------|------|
| 최신(1년 이내) 서류 제출 | 3.33 | 법인등기부등본, 사업자등록증 등 최신 서류 |
| 서류 제출 (1년 초과) | 6.66 | 서류를 제출하였으나 최신이 아닌 경우 |
| 서류 제출 불가 | 10 | 서류를 제출할 수 없는 경우 |

- **조사 방법**: 프로젝트 측에 서류 제출 요청 → 제출 여부 확인
- **AI 자동화**: 불가 (외부 제출 의존)
- **실무 참고**: 대부분의 해외 프로젝트는 "서류 제출 불가(10점)"에 해당

#### 항목 2) 실사보고서 (별도 체크, 3.33 또는 10점)

| 선택지 | 점수 | 설명 |
|--------|------|------|
| 보고서 제출 가능 & 결함 없음 | 3.33 | 감사보고서, 기술보고서, 보안감사 등 |
| 미제출 또는 결함 발견 | 10 | 보고서가 없거나 결함이 있는 경우 |

- **조사 방법**: 프로젝트의 GitHub에서 보안 감사 보고서 검색, 공식 문서 확인
- **AI 자동화**: 부분 가능 (GitHub audit 보고서 링크 검색)
- **예시**:
  - Recall: Sigma Prime 보안 감사 보고서 → True (3.33점)
  - B3: Quantstamp 감사 인증 → True (3.33점)
  - AVNT: 보고서 없음 → False (10점)

#### 항목 3) 국가 정보 (0~10점, 국가위험점수표 참조)

| 하위 항목 | 산출 방법 |
|----------|----------|
| 국적의 국가위험점수 | 국가위험점수표에서 VLOOKUP |
| 소재지의 국가위험점수 | 국가위험점수표에서 VLOOKUP |
| 관할권의 국가위험점수 | 국가위험점수표에서 VLOOKUP |
| **최종 점수** | **3개 중 MAX** |

- **조사 방법**: 섹션 2-3에서 파악한 국가를 국가위험점수표에서 조회
- **AI 자동화**: 완전 자동화 가능 (국가위험점수 테이블 내장)
- **예시**:
  - 버진아일랜드 → 9.17
  - 파나마 → 8.89
  - 홍콩 → 5.55
  - 대한민국 → 3.33

#### 항목 4) 대표자 정보 (택1, 1~10점)

| 선택지 | 점수 | 설명 |
|--------|------|------|

| 2 | 신규 가상자산과 관련된 거래를 모니터링하기 위한 룰을 준비하였으며, 시뮬레이션을 통해 적정성 여부를 확인하였는가? | 이상거래 모니터링 룰 셋업 및 테스트 완료 여부 |

### 3.6 섹션 5: 종합의견

#### 5-1) 주관부서 종합의견

주관부서(사업운영팀/플랫폼운영팀)가 작성하는 운영 관점의 의견이다.

**작성 패턴** (기존 사례 기반):

```
기술적 표준성: [네트워크] 네트워크에서 동작하며, 입출고 관리상 특이사항 없음.
공시 관리: 공식 홈페이지 및 소셜 채널을 통해 사업 현황을 지속적으로 업데이트하고 있어 운영 투명성이 양호함.
리스크 이력 부재: 최근 3년 내 해킹, 자금세탁 등 보안 사고나 법적 이슈가 발생한 이력이 없어 금융 사고 위험성이 낮은 것으로 판단됨.
```

- 위 3가지 항목을 기본 골격으로 작성
- 특이사항이 있으면 추가 기술 (예: 고레버리지 구조, 특수한 토큰 구조 등)
- **AI 자동화**: 완전 자동화 가능 (리서치 결과 기반 초안 생성)

#### 5-2) 전담부서 종합의견

AML팀이 작성하는 자금세탁방지 관점의 종합 평가 의견이다.

**작성 구조** (기존 사례 기반):

```
1단락: [자산명]은 [발행법인]이 발행·운영하는 가상자산으로, [네트워크] 네트워크를 기반으로 [프로젝트 설명]. 발행 법인의 관할국은 [국가]로 확인되며, [FATF 관련 언급].

2단락: 가상자산 거래지원 적격성 평가 결과, [자산명]은 다크코인 또는 프라이버시 강화 코인에 해당하지 않으며, 제재 대상 또는 고위험 국가와의 직접적 연관성도 확인되지 않음. 이에 따라 거래지원 적격성 평가 결과는 거래지원 가능으로 판단됨.

3단락: 가상자산 고유위험 평가 결과, [법인 정보 확인 수준], [대표자 정보 확인 수준], [부정적 이슈 유무], [거래 추적 가능성]. 종합 고유위험 점수는 [X.XX]점으로, 전반적으로 관리 가능한 수준의 위험도로 평가됨.

4단락: 가상자산 통제효과성 평가 결과, 입·출금 주소 관리 체계, 거래 추적 가능성, 내부 모니터링 및 시뮬레이션 테스트 수행 여부 등 주요 통제 항목에서 적정 수준의 통제 체계가 구축된 것으로 확인되며 평가 결과는 양호로 판단됨.

5단락: 종합적으로, [자산명]은 [위험 요소 요약], 거래지원 적격성, 통제효과성 및 현재까지 확인된 리스크 요인을 종합적으로 고려할 때 현 시점에서 거래지원을 제한할 중대한 사유는 없는 것으로 판단됨.

6단락(표준): 아울러, 본 평가는 커스터디 서비스 제공을 전제로 수행된 것으로, 직접적인 거래 개입이나 가격 형성 과정에 관여하지 않는 구조임을 감안할 때, 상대적으로 제한된 위험 환경에서 관리 가능한 수준의 서비스 제공이 가능할 것으로 판단됨.
```

- **AI 자동화**: 초안 생성 가능 (최종 확정은 AML팀 리뷰 필수)

## 4. 점수 산출 로직

### 4.1 고유위험 점수 산출 수식

```
총점 = 법인정보 + 실사보고서 + 국가위험 + 대표자정보
     + 이슈(자산) + 이슈(대표자) + 이슈(재단)
     + 스테이블코인 + 거래소상장 + 거래추적
```

각 항목별 상세:

```
법인정보     = 최신서류(3.33) | 서류(6.66) | 불가(10)           ← 택1
실사보고서   = 제출가능&정상(3.33) | 미제출(10)                 ← 별도체크
국가위험     = MAX(국적점수, 소재지점수, 관할권점수)             ← 0~10
대표자정보   = 신분증(1) | 정보제출(3.33) | 검색가능(6.66) | 불가(10) ← 택1
이슈(자산)   = 있음(10) | 없음(3.33)
이슈(대표자) = 있음(10) | 없음(3.33)
이슈(재단)   = 있음(10) | 없음(3.33)
스테이블코인 = 예(10) | 아니오(3.33)
거래소2곳+   = 2곳이상(3.33) | 미만(10)
거래추적     = 지원(3.33) | 미지원(10)
```
