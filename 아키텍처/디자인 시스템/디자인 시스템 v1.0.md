# WorkHub 통합 디자인 시스템 — B3: Warm Elevated

> Apple/Raycast 스타일. 테두리 없이 그림자만으로 카드를 분리하는 프리미엄 라이트 모드 디자인.
> 모든 허브(HubManager, SalesHub, FinHub, StrategyHub 등)에 공통 적용.

---

## 1. 디자인 철학

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **Elevated** | 카드가 배경 위에 떠있는 듯한 부유감. 테두리 대신 그림자로 구분 |

## 5. 그림자 시스템 (핵심)

B3의 핵심: 그림자를 주된 분리 수단으로 사용하되, **데이터 밀집 화면에서는 하이브리드(그림자+테두리) 적용**.

### 5.0 하이브리드 전략 (Shadow + Border)

순수 shadow-only는 KPI 카드, 대시보드 등 **카드 중심 화면**에 최적.
테이블, 폼, 상세 패널 등 **데이터 밀집 화면**에서는 행 경계가 불명확해지므로 하이브리드 사용.

| 컴포넌트 | 전략 | 이유 |
|---------|------|------|
| KPI/통계 카드 | `shadow-md` only | 부유감, 시각적 임팩트 |
| 테이블 카드 | `shadow-sm` + `border border-zinc-200` | 행 경계 명확성 |
| 폼/입력 영역 | `border` + 강한 focus ring | 입력 영역 구분 |
| 상세 패널 내부 | `divide-y` (구분선) > shadow | 정보 그룹핑 |
| 모달/팝오버 | `shadow-xl` only | 오버레이와 함께 충분 |
| 사이드바 | `shadow-2xl` only | 고정 요소 |

### 5.1 그림자 스케일

| 토큰 | CSS 값 | Tailwind | 용도 |
|------|--------|----------|------|
| `--shadow-xs` | `0 1px 2px rgba(0,0,0,0.04)` | `shadow-xs` | 버튼, 입력 필드 |
| `--shadow-sm` | `0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)` | `shadow-sm` | 작은 카드, 드롭다운 |
| `--shadow-md` | `0 4px 12px rgba(0,0,0,0.08), 0 1px 3px rgba(0,0,0,0.04)` | `shadow-md` | **기본 카드 (기본값)** |
| `--shadow-lg` | `0 8px 24px rgba(0,0,0,0.10), 0 2px 6px rgba(0,0,0,0.04)` | `shadow-lg` | **카드 호버 상태** |
| `--shadow-xl` | `0 16px 40px rgba(0,0,0,0.12), 0 4px 12px rgba(0,0,0,0.04)` | `shadow-xl` | 모달, 팝오버 |
| `--shadow-2xl` | `0 24px 56px rgba(0,0,0,0.16)` | `shadow-2xl` | 사이드바, AI 패널 |

### 5.2 사용 규칙

```
카드 기본:     shadow-md  →  호버: shadow-lg
사이드바:      shadow-2xl (고정 요소)
모달:          shadow-xl + overlay
드롭다운:      shadow-lg
버튼:          shadow-xs  →  호버: shadow-sm
입력 필드:     shadow-xs  →  포커스: ring-2 ring-primary-700/20
```

### 5.3 테두리 규칙

| 상황 | 테두리 | 설명 |
|------|--------|------|
| **KPI/대시보드 카드** | 없음 (`border-0`) | 그림자만으로 분리 |
| **테이블 카드** | `border border-zinc-200` | 데이터 밀집 → 하이브리드 |
| **입력 필드** | `border zinc-300` | 입력 영역 구분 필요 |
| **테이블 행** | `divide-y zinc-100` | 행 간 미세 구분 |
| **사이드바** | 없음 (`border-0`) | 그림자로 분리 |
| **구분선** | `zinc-100` | 섹션 내 구분 |

### 7.3 배지 시스템 v2 (3단계)

배지는 정보의 **중요도**에 따라 3단계로 구분. 모든 배지를 동일한 솔리드로 처리하면 시각적 피로 발생.

#### 단계 정의

| 단계 | 용도 | 스타일 | 예시 |
|------|------|--------|------|
| **L1 — Critical** | 리스크, 차단, 컴플라이언스 | 진한 솔리드 + 아이콘 | Drop, 블록됨, 긴급 |
| **L2 — Status** | 상태, 진행 단계 | 소프트 솔리드 (연한 배경 + 진한 텍스트) | 액티브, 진행중, 대기 |
| **L3 — Meta** | 분류, 타입, 카테고리 | 아웃라인 또는 subtle | 법인, 금융, 펀드 |

> **규칙**: L3 메타데이터에 고채도 솔리드 색상 사용 금지.

#### L1 — Critical 배지

```html
<span class="
  inline-flex items-center gap-1
  px-2.5 py-1
  rounded-full
  text-xs font-semibold
  bg-red-600 text-white
">
  <AlertTriangleIcon class="w-3 h-3" />
  Drop
</span>
```

#### L2 — Status 배지

```html
<!-- 활성 상태 (소프트 솔리드) -->
<span class="
  inline-flex items-center
  px-2.5 py-1
  rounded-full
  text-xs font-medium
  bg-green-100 text-green-800
">
  액티브 고객
</span>

<!-- 진행중 -->
<span class="... bg-blue-100 text-blue-800">회원가입 진행중</span>

<!-- 대기 -->
<span class="... bg-amber-100 text-amber-800">대기중</span>
```

#### L3 — Meta 배지

```html
<!-- 아웃라인 스타일 -->

  inline-flex items-center
  px-2.5 py-1
  rounded-full
  text-xs font-medium
  border border-zinc-300 text-zinc-600 bg-transparent
">
  금융
</span>

<!-- 법인 구분 -->
<span class="... border border-indigo-300 text-indigo-700 bg-indigo-50">법인</span>
```

### 7.5 테이블 시스템 v2

#### Density 모드

| 모드 | 행 높이 | 셀 폰트 | 카드 패딩 | 사용 화면 |
|------|---------|---------|----------|----------|
| **Default** | 52-56px | 13px | `p-8` | 대시보드 내 요약 테이블 |
| **Dense** | 40-44px | 12px | `p-6` | **고객 리스트, 로그, 재무 테이블** |

> Dense 모드는 고객 리스트, 활동 로그, 재무 테이블에 **필수 적용**.

#### 전환 방식

```html
<!-- data attribute로 density 제어 -->
<div data-density="dense" class="...">
  <table class="w-full data-[density=dense]:text-xs">
    ...
  </table>
</div>
```

또는 화면 유형별 고정:
- 고객 리스트 → Dense (항상)
- 대시보드 최근 목록 → Default
- 재무 테이블 → Dense (항상)

#### 숫자 규칙 (핀테크 필수)

| 규칙 | 설명 | 예시 |
|------|------|------|
| 정렬 | 숫자 열은 **우측 정렬** | `text-right` |
| 폰트 | `font-mono` + `tabular-nums` | 열 정렬 유지 |
| 단위 분리 | 숫자와 단위를 분리, 단위는 muted | `22` + `<span class="text-zinc-400">명</span>` |
| 소수점 | 퍼센트 = 소수 1자리, 금액 = 정수 | `12.5%`, `₩138,500,000` |
| 델타 | 부호 항상 표시 (+ / −) | `+12.5%`, `−3.2%` |
| 금액 | 쉼표 구분 + 통화기호 | `₩138,500,000` |

## 13. 필터 바 v1

### 구조

```
데스크톱:
┌────────────────────────────────────────────────────────────┐
│ [🔍 검색]  [단계 ▼]  [법인구분 ▼]  [회신차례 ▼]    [+ 필터]  [새 고객] │
│ [활성 필터 칩: ✕ 액티브 고객] [✕ 법인]                              │
└────────────────────────────────────────────────────────────┘

모바일:
┌──────────────────────────┐
│ [🔍 검색]       [필터 ⚙️]  │
│ [✕ 액티브] [✕ 법인]  →    │
└──────────────────────────┘
```

### 규칙

- 기본 1줄 레이아웃, **활성 필터가 있을 때만** 칩 행 표시
- 필터 3개 이상이면 `+ 필터` 버튼으로 확장 행 제공
- 모바일: 검색만 노출, 필터는 `필터` 버튼 → 바텀시트/드로어
- 활성 필터 칩: 가로 스크롤 (`overflow-x-auto flex gap-2`)

### 필터 칩 스타일

```html
<!-- 활성 필터 칩 -->

```

### 모바일 필터 드로어


## 14. KPI 카드 표준

### 필수 요소

모든 KPI 카드는 다음 4가지를 반드시 포함:

| 요소 | 설명 | 위치 |
|------|------|------|
| **주요 값** | 핵심 수치 (큰 텍스트) | 중앙 |
| **델타** | 전주/전월 대비 변화율 | 주요 값 옆 또는 아래 |
| **기준 기간** | "2월 기준", "이번 주" 등 | 제목 옆 또는 아래 |
| **데이터 기준 시간** | 마지막 갱신 시점 | 카드 하단 (미세 텍스트) |

> **규칙**: 컨텍스트 없는 KPI 카드 금지.

### 구현

```html
<div class="bg-white rounded-2xl shadow-md p-8">
  <!-- 헤더 -->
  <div class="flex items-center justify-between mb-1">
    <span class="text-sm font-medium text-zinc-500">월간 거래액</span>
    <span class="text-xs text-zinc-400">2월 기준</span>
  </div>

  <!-- 주요 값 + 델타 -->
  <div class="flex items-baseline gap-3 mb-1">
    <span class="text-3xl font-bold text-zinc-900 font-mono tabular-nums">
      ₩138,500,000
    </span>
    <span class="inline-flex items-center text-sm font-medium text-green-700">
      <TrendingUpIcon class="w-3.5 h-3.5 mr-0.5" />
      +12.5%
    </span>
  </div>

  <!-- 비교 대상 -->
  <p class="text-xs text-zinc-400">vs 전월 ₩123,100,000</p>

  <!-- 데이터 기준 시간 -->
  <p class="text-[11px] text-zinc-300 mt-4">기준: 2/7 09:00 KST</p>
</div>
```
