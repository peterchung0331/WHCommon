# 계정 관리 아키텍처

## 개요

WorkHub의 계정 관리는 **HubManager에서 중앙 집중 관리**됩니다. 각 허브(SalesHub, FinHub 등)에는 계정 관리 기능이 없으며, HubManager 어드민이 모든 계정 생성, 권한 부여, 비밀번호 초기화를 담당합니다.

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **중앙 관리** | 계정 CRUD는 HubManager `/admin/accounts`에서만 수행 |
| **승인 절차 없음** | 어드민이 직접 계정 생성 + 임시 비밀번호 발급 |
| **허브별 인증** | 각 허브는 자체 세션으로 로그인 (HubManager DB 참조) |
| **허브별 권한** | hub_memberships 테이블로 허브별 역할 할당 |

---

## 역할 체계

### 5단계 역할 (role_level 기준, 낮을수록 높은 권한)

| 레벨 | 역할 | 설명 | 데이터 접근 |
|------|------|------|------------|
| 1 | **MASTER** | 최고 관리자 | 전체 + 계정 관리 |
| 2 | **ADMIN** | 관리자 | 전체 + 설정 변경 |
| 3 | **USER** | 일반 사용자 | 전체 (읽기/쓰기) |
| 4 | **VIEWER** | 읽기 전용 | 연락처 부분 마스킹 |
| 5 | **GUEST** | 게스트 | 전체 마스킹 + 제한된 페이지 |

### 역할별 권한 매트릭스

| 기능 | MASTER | ADMIN | USER | VIEWER | GUEST |
|------|--------|-------|------|--------|-------|
| 데이터 조회 | O | O | O | O (마스킹) | O (전체 마스킹) |
| 데이터 생성/수정 | O | O | O | X | X |
| 데이터 삭제 | O | O | X | X | X |
| 설정 변경 | O | O | X | X | X |
| 계정 관리 | O (HubManager) | O (HubManager) | X | X | X |

## 계정 생명주기

```
어드민이 HubManager에서 계정 생성
    │
    ├─ 이메일 + 풀네임 입력
    ├─ 허브별 권한 할당 (선택)
    └─ 임시 비밀번호 자동 생성 (8자, 영문+숫자)
         │
         ▼
사용자에게 임시 비밀번호 전달 (수동)
         │
         ▼
사용자가 허브에서 첫 로그인
    │
    ├─ 임시 비밀번호로 인증
    ├─ must_change_password = true → 비밀번호 변경 페이지로 리다이렉트
    └─ 새 비밀번호 설정 (8자 이상, 영문+숫자 필수)
         │
         ▼
정상 사용 (ACTIVE)
    │
    ├─ 비밀번호 초기화 (어드민) → 새 임시 비밀번호 발급
    ├─ 계정 비활성화 (어드민) → 로그인 차단
    └─ 권한 변경 (어드민) → hub_memberships 업데이트
```

## DB 스키마

### users 테이블 (HubManager DB)

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  username VARCHAR(100) UNIQUE NOT NULL,      -- 이메일 @ 앞부분 자동 추출
  password_hash VARCHAR(255) NOT NULL,        -- bcrypt (salt: 10)
  full_name VARCHAR(255),
  is_admin BOOLEAN DEFAULT false,
  is_active BOOLEAN DEFAULT true,
  status VARCHAR(20) DEFAULT 'active',        -- active, inactive
  must_change_password BOOLEAN DEFAULT true,  -- 임시 비밀번호 변경 필요
  password_changed_at TIMESTAMP,
  last_login_at TIMESTAMP,
  created_by INTEGER REFERENCES users(id),    -- 계정 생성자
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### hub_memberships 테이블 (HubManager DB)

```sql
CREATE TABLE hub_memberships (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id),
  hub_id INTEGER NOT NULL REFERENCES hubs(id),
  role VARCHAR(50) NOT NULL,                  -- MASTER, ADMIN, USER, VIEWER
  status VARCHAR(20) DEFAULT 'ACTIVE',        -- ACTIVE, INACTIVE, SUSPENDED
  granted_by INTEGER REFERENCES users(id),
  granted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  revoked_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, hub_id)
);
```

### hub_role_definitions 테이블 (HubManager DB)

```sql
CREATE TABLE hub_role_definitions (
  id SERIAL PRIMARY KEY,
  hub_id INTEGER NOT NULL REFERENCES hubs(id),
  role_name VARCHAR(50) NOT NULL,             -- MASTER, ADMIN, USER, VIEWER, GUEST
  role_display_name VARCHAR(100) NOT NULL,
  role_description TEXT,
  permissions JSONB DEFAULT '{}',
  is_default BOOLEAN DEFAULT false,
  role_level INTEGER DEFAULT 99,              -- 1=MASTER, 2=ADMIN, 3=USER, 4=VIEWER, 5=GUEST
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(hub_id, role_name)
);
```

### audit_logs 테이블 (HubManager DB)

```sql
CREATE TABLE audit_logs (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  hub_id INTEGER REFERENCES hubs(id),
  action VARCHAR(50) NOT NULL,                -- LOGIN, PERMISSION_GRANTED, ROLE_CHANGED 등
  entity_type VARCHAR(50),
  entity_id VARCHAR(255),
  old_value JSONB,
  new_value JSONB,
  ip_address VARCHAR(45),
  user_agent TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### accounts 테이블 (각 허브 DB)

각 허브는 자체 `accounts` 테이블을 갖되, HubManager의 users 테이블과 동기화됩니다.

```sql
CREATE TABLE accounts (
  id UUID PRIMARY KEY,
  account_id VARCHAR(255) UNIQUE,             -- username (HubManager users.username과 동일)
  email VARCHAR(255) UNIQUE,
  password_hash VARCHAR(255),
  name VARCHAR(255),
  role VARCHAR(20) NOT NULL,                  -- hub_memberships.role에서 가져옴
  status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE',
  must_change_password BOOLEAN DEFAULT false,
  last_login_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

## HubManager 어드민 API

### 어드민 인증

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/admin/auth` | 어드민 비밀번호 인증 (세션에 플래그 저장) |
| GET | `/api/admin/check` | 어드민 인증 상태 확인 |
| POST | `/api/admin/logout` | 어드민 로그아웃 |

### 계정 관리

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/admin/accounts` | 전체 사용자 목록 |
| POST | `/api/admin/accounts/create` | 계정 생성 (임시 비밀번호 반환) |
| PATCH | `/api/admin/accounts/:id/reset-password` | 비밀번호 초기화 (새 임시 비밀번호) |
| PATCH | `/api/admin/accounts/:id/toggle-active` | 계정 활성/비활성 토글 |
| POST | `/api/admin/accounts/change-password` | 비밀번호 변경 (사용자 본인) |

### 허브 권한 관리

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/admin/accounts/:id/hub-permissions` | 사용자 허브 권한 조회 |
| POST | `/api/admin/accounts/:id/hub-permissions` | 허브 권한 부여/변경 |
| DELETE | `/api/admin/accounts/:id/hub-permissions/:hubId` | 허브 권한 취소 |
| GET | `/api/admin/hubs/:hubId/role-definitions` | 허브별 역할 정의 조회 |
| GET | `/api/auth/verify-hub-permission` | 허브 권한 검증 (허브에서 호출) |

### 감사 로그

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/admin/audit-logs` | 전체 감사 로그 |
| GET | `/api/admin/audit-logs/hub/:hubSlug` | 허브별 감사 로그 |
| POST | `/api/internal/audit-logs` | 감사 로그 생성 (허브에서 호출) |

## 인증 흐름 (허브별)

### SalesHub (세션 기반)

```
사용자 → /login 접근
    │
    ├─ POST /api/auth/login (email + password)
    │  ├─ HubManager DB에서 사용자 조회
    │  ├─ bcrypt 비밀번호 검증
    │  ├─ hub_memberships에서 SalesHub 권한 확인
    │  ├─ express-session 세션 생성 (PostgreSQL 저장)
    │  └─ 쿠키 설정 (wbhub.sid)
    │
    ├─ must_change_password = true → /change-password 리다이렉트
    │
    └─ 이후 API 호출 → 세션 쿠키 자동 전송 → 인증 확인
```

### 게스트 입장 (SalesHub 전용)

```
비로그인 사용자 → 게스트 입장 버튼 클릭
    │
    ├─ POST /api/auth/guest
    │  ├─ 임시 세션 생성 (GUEST 역할)
    │  ├─ 1시간 TTL
    │  └─ 쿠키 설정
    │
    └─ 제한된 데이터 접근 (전체 마스킹)
```

## 허브별 구현 규칙

### 허용됨 (각 허브)

- 자체 로그인/로그아웃 (세션 기반)
- `/api/auth/me` - 현재 사용자 정보 반환
- `/api/auth/login` - 로그인 (HubManager DB 참조)
- `/api/auth/logout` - 로그아웃
- `/api/auth/change-password` - 비밀번호 변경
- `/api/auth/guest` - 게스트 입장 (SalesHub만)
- 역할 기반 미들웨어 (`isAuthenticatedAndActive`, `hasMinRole`)
- 데이터 마스킹 미들웨어 (`applyDataMasking`)

### 금지됨 (각 허브)

- 계정 CRUD API (`/api/accounts` 엔드포인트)
- 계정 관리 프론트엔드 페이지 (`/accounts`)
- 계정 승인/거부 기능
- 권한 부여/취소 기능
- 비밀번호 초기화 기능 (어드민만 HubManager에서 가능)
- 계정 목록 조회 API

## 프론트엔드 (HubManager /admin/accounts)

### 기능

1. **사용자 목록**: 전체 등록 사용자 조회 (이메일, 이름, 상태, 마지막 로그인)
2. **계정 생성**: 이메일 + 풀네임 입력 → 임시 비밀번호 자동 생성 → 화면에 표시
3. **허브 권한 관리**: 사용자별 허브 접근 권한 부여/변경/취소
4. **비밀번호 초기화**: 임시 비밀번호 재발급
5. **계정 활성/비활성**: 계정 상태 토글
6. **감사 로그**: 권한 변경 이력 조회

### 접근 제어

- 어드민 비밀번호 별도 인증 (세션 내 `isAdminAuthenticated` 플래그)
- HubManager 로그인 + 어드민 비밀번호 2단계 인증
