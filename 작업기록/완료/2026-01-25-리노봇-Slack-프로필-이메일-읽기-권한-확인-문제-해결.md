## 문제 진단

### 핵심 원인
리노봇이 Slack 프로필에서 이메일을 읽지 못하는 이유는 **Slack App OAuth Scope 미설정**

- ❌ `users:read` scope 없음 → Slack 사용자 정보 조회 불가
- ❌ `users:read.email` scope 없음 → 이메일 필드 접근 불가
- 결과: `users.info()` API 호출 실패 → WBSalesHub 계정 매칭 실패 → 권한 확인 불가

### 현재 상태 (작업 전)
- ✅ WBSalesHub의 권한 확인 로직은 완벽하게 구현됨 ([renoSlackApp.ts:60-158](WBSalesHub/server/modules/integrations/slack/renoSlackApp.ts#L60-L158))
- ✅ Slack API 호출 코드도 이미 존재함
- ✅ `@slack/bolt` 패키지 설치됨
- ❌ **OAuth Scope만 추가하면 즉시 작동**

```typescript
/**
 * Determine persona variant based on user permission
 * - Internal: 직원 (ADMIN + hasCustomerAccess)
 * - External: 고객 또는 권한 없는 사용자
 */
private determinePersonaVariant(permission: UserPermission): PersonaVariant {
  // ADMIN 역할이고 고객 접근 권한이 있으면 Internal
  if (permission.hasCustomerAccess && permission.user?.role === 'ADMIN') {
    return 'internal';
  }

  // 그 외 모든 경우 External (고객 대응 톤)
  return 'external';
}
```
