## 📋 작업 개요

Reno Slack 봇의 헬프 메시지를 DB에 저장하여 관리할 수 있도록 개선했습니다.

### 사용자 요구사항
1. **두 가지 메시지로 분류**
   - 대화형 기능: 슬래시 명령어, AI Tool Use
   - 자동화 기능: 크론 스케줄러 작업
2. **DB 테이블에 저장**
   - 코드 배포 없이 메시지 수정 가능
   - 관리자가 API로 업데이트 가능
3. **자연어 트리거 지원**
   - "뭘 할 수 있어?", "기능", "help", "도움말" 등

---

## ✅ 구현 완료 내역

### 1. DB 스키마 (Migration 016)

**파일**: `server/database/migrations/016_add_bot_help_messages.sql`

**테이블 구조**:
```sql
CREATE TABLE bot_help_messages (
  id SERIAL PRIMARY KEY,
  message_type VARCHAR(50) NOT NULL,  -- 'interactive' | 'automated'
  title TEXT NOT NULL,
  content JSONB NOT NULL,             -- Slack Block 구조
  plain_text TEXT,                    -- 플레인 텍스트 fallback
  enabled BOOLEAN DEFAULT true,
  display_order INTEGER DEFAULT 0,
  version INTEGER DEFAULT 1,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_by VARCHAR(255)
);
```

**초기 데이터**:
- 메시지 1 (interactive): 대화형 기능 안내
- 메시지 2 (automated): 자동화 기능 안내

### 2. 서비스 레이어

**파일**: `server/services/botHelpMessageService.ts`

**주요 함수**:
- `getEnabledHelpMessages()`: 활성화된 메시지 조회
- `getHelpMessageByType()`: 타입별 메시지 조회
- `updateHelpMessage()`: 관리자용 메시지 업데이트

### 3. API 엔드포인트

**파일**: `server/routes/botHelpMessageRoutes.ts`

**엔드포인트**:
- `GET /api/bot-help-messages`: 메시지 목록 조회
- `PUT /api/bot-help-messages/:id`: 메시지 업데이트 (관리자 전용)

### 4. Slack 통합

**파일**: `server/modules/integrations/slack/renoSlackApp.ts`

**개선 사항**:
1. `/reno-help` 명령어: DB에서 메시지 로드
2. 자연어 트리거: `@Reno 뭘 할 수 있어?` 감지

**자연어 키워드**:
```typescript
const helpKeywords = ['뭘 할 수 있', '기능', 'help', '도움말', '사용법'];
```

### 개선 기술 (웹 검색 결과)

#### 1. Intent Classification (LLM 기반)
**출처**: [Building a ChatOps AI Bot with LangChain and LLMs in Slack](https://opsinsights.dev/building-a-chatops-ai-bot-with-langchain-and-llms-in-slack)

- LLM이 자연어를 구조화된 JSON으로 변환
- Intent classifier가 유효한 명령만 실행 (환각 방지)
- 신뢰도(confidence) 기반 에러 핸들링

#### 2. Claude Advanced Tool Use
**출처**: [Anthropic - Advanced Tool Use](https://www.anthropic.com/engineering/advanced-tool-use)

- **Programmatic Tool Calling**: Claude가 코드를 작성하여 여러 도구를 순차 실행
- **Tool Search Tool**: 필요한 도구만 동적으로 발견 (MCP 서버처럼)

#### 3. Named Entity Recognition (NER)
**출처**: [Named Entity Recognition with LLMs](https://medium.com/@grisanti.isidoro/named-entity-recognition-with-llms-extract-conversation-metadata-94d5536178f2)

- 자연어 지시문으로 임의의 엔티티 추출
- 도메인별 커스터마이징 쉬움
- [chatbot_ner 라이브러리](https://github.com/hellohaptik/chatbot_ner) 활용

#### 4. Multi-turn Context Retention
**출처**: [Claude Advanced Guide](https://www.graphapp.ai/blog/claude-advanced-guide-mastering-the-ai-language-model)

- 이전 대화 이력을 계속 전달
- 사용자가 "그 고객"이라고 해도 ABC123 자동 인식

#### 5. Golden Conversations
**출처**: [Prototyping Conversational AI Flows](https://www.chatprd.ai/how-i-ai/workflows/how-to-prototype-conversational-ai-flows-using-claude-s-golden-conversations)

- 이상적인 대화 샘플 작성
- Claude에게 시스템 프롬프트 생성 요청
- 즉시 프로토타입 테스트 가능
