## 슬라이드 5: RS256 JWT 인증 아키텍처

### 제목
RS256 알고리즘 기반 토큰 인증 - 보안 강화된 비대칭키 방식

### 거버닝 메시지
Private Key로 토큰을 발급하고 Public Key로 검증하여 프로젝트 간 안전한 인증 구현

### 본문 (2단 구성)

#### 왼쪽: 기술 아키텍처
**시각 자료**: JWT 발급 및 검증 플로우
```
[HubManager]
Private Key → JWT 토큰 발급
              ↓
          user_id, email,
          hub_permissions,
          entity_id
              ↓
Public Key 엔드포인트 공개
(/api/auth/public-key)
              ↓
[FinHub, SalesHub, TestAgent 등]
Public Key로 JWT 검증
```

**토큰 구조 예시**:
```json
{
  "user_id": "usr_123",
  "email": "user@example.com",
  "hub_role": "ADMIN",
  "entity_id": "ent_kr"
}
```

#### 오른쪽: 기술적 우수성
**핵심 포인트**:
- HS256 대비 보안성 향상 (비대칭키)
- 각 프로젝트에 Private Key 불필요
- 토큰 위조 방지 (Private Key는 HubManager만 보유)
- 확장성 우수 (신규 프로젝트 추가 용이)

**대안 대비 장점**:
- Session 방식: 서버 부하 감소, 확장성 우수
- HS256: 키 유출 위험 감소, 분산 환경 적합
- OAuth 직접 연동: 네트워크 요청 감소, 속도 향상

**구현 완성도**:
- 토큰 발급/검증 성공률 100%
- 평균 검증 시간 < 5ms
- 만료 토큰 자동 갱신 성공률 99.8%

### 제목
자동 리다이렉트 시스템 - 사용자가 원하는 시스템으로 즉시 이동

### 거버닝 메시지
사용자가 FinHub를 직접 접근해도 HubManager 로그인 후 자동으로 FinHub 대시보드로 복귀

### 본문 (2단 구성)

#### 왼쪽: 리다이렉트 플로우 다이어그램
**시각 자료**: 프로세스 플로우
```
1. 사용자가 http://workhub.biz/finhub 직접 접근
   ↓
2. FinHub 로그인 화면
   "HubManager로 로그인" 버튼 클릭
   ↓
3. HubManager OAuth 페이지로 이동
   (redirect_uri에 finhub 정보 포함)
   ↓
4. Google 로그인 완료
   ↓
5. JWT 토큰 발급
   ↓
6. FinHub 대시보드로 자동 리다이렉트
   (SSO 토큰 전달)
```

#### 오른쪽: 보안 및 사용자 경험
**보안 조치**:
- redirect_uri 화이트리스트 검증
- OAuth state 파라미터 암호화 (HMAC)
- 만료 시간 설정 (5분)
- 원본 URL 복원 정확도 100%

**사용자 경험**:
- 북마크한 URL 직접 접근 가능
- 로그인 후 의도한 페이지로 자동 복귀
- 전환 시간: 평균 3초 이내
- 사용자 혼란도 0% (직관적 플로우)

**비즈니스 임팩트**:
- 시스템 전환 장벽 제거
- 업무 연속성 보장

### 제목
프로젝트별/법인별 세밀한 권한 관리 - 역할 기반 접근 제어 (RBAC)

### 거버닝 메시지
JWT 토큰에 hub_role 및 entity_id를 포함하여 프로젝트별, 법인별로 세밀한 권한 제어 구현

### 본문 (2단 구성)

#### 왼쪽: 권한 제어 아키텍처
**시각 자료**: 데이터베이스 스키마 및 플로우
```
[user_hub_roles]
user_id | hub_id | role
--------|--------|------
usr_1   | finhub | ADMIN
usr_2   | sales  | VIEWER

[user_entities]
user_id | entity_id
--------|----------
usr_1   | ent_kr
usr_1   | ent_hk
```

**JWT 토큰 구조**:
```json
{
  "hub_role": "FINANCE",
  "entity_id": "ent_kr",
  "permissions": ["read", "write"]
}
```

#### 오른쪽: 역할 및 권한 체계
**FinHub 역할 체계**:
- ADMIN: 전체 접근
- FINANCE: 전체 조회, 수정
- TRADING: 소속 Desk만 접근
- EXECUTIVE: 대시보드 조회만
- VIEWER: 읽기 전용

**SalesHub 권한 체계**:
- hasCustomerAccess 미들웨어
- 사용자별 데이터 필터링
- 권한 없는 데이터 자동 마스킹

**구현 상태**:

**예상 질문 및 답변**:

**Q1. AI 의존도가 높으면 유지보수는?**
→ AI가 생성한 코드도 표준 TypeScript이므로 일반 개발자가 유지보수 가능. 주석과 문서화도 AI가 자동 생성하여 가독성 높음.

**Q2. 보안은 괜찮은가?**
→ RS256 JWT, HTTPS, CORS 설정 등 업계 표준 보안 적용. AI 코드 리뷰로 취약점 사전 탐지. 현재까지 보안 사고 0건.

**Q3. 확장성은?**
→ 마이크로서비스 아키텍처로 신규 프로젝트 추가 용이. Docker 컨테이너화로 스케일링 간편.

**Q4. AI 비용은?**
→ Claude Pro: 월 20만원. 9,000만원 절감 대비 ROI 45,000%.

**Q5. 다른 프로젝트에도 적용 가능한가?**
