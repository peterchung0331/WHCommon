# 스킬테스터 (메인 디스패처)

## 개요
자연어 명령을 파싱하여 테스트 유형(단위/통합/E2E)을 자동 분류하고,
해당하는 서브 스킬을 로드하여 테스트를 실행합니다.

## 실행 조건
- `/스킬테스터` 또는 `/스킬테스터 [명령]` 호출 시

## 서브 스킬
| 스킬 | 파일 | 용도 |
|------|------|------|
| 스킬테스터-단위 | `스킬테스터-단위.md` | Jest/Vitest 단위 테스트 |
| 스킬테스터-통합 | `스킬테스터-통합.md` | API/Supertest 통합 테스트 |
| 스킬테스터-E2E | `스킬테스터-E2E.md` | Playwright E2E 테스트 |
| 스킬테스터-스모크 | `스킬테스터-스모크.md` | API 스모크 테스트 (필수 체크) |

## 자연어 파싱 규칙

### 테스트 유형 키워드
| 키워드 | 유형 | 호출 스킬 |

| "단위", "유닛", "unit" | Unit | `스킬테스터-단위` |
| "통합", "integration", "API" | Integration | `스킬테스터-통합` |
| "E2E", "이투이", "end-to-end", "브라우저" | E2E | `스킬테스터-E2E` |

### 화살표 (→) 의미
- `A → B` = A에서 시작하여 B로 이동하는 플로우 테스트
- E2E 또는 통합 테스트에서 사용

## 작업 순서

### 1단계: 명령 파싱
```
입력: "오라클에서 허브매니저 -> 세일즈허브를 E2E 테스트 해줘"

파싱 결과:
├─ 환경: oracle
├─ 시작점: WBHubManager
├─ 목표: WBSalesHub
├─ 유형: E2E
└─ 호출: 스킬테스터-E2E
```

### 2단계: 서브 스킬 호출
- 파싱된 정보를 서브 스킬에 전달
- 서브 스킬이 실제 테스트 실행

### 3단계: 결과 수집
- 서브 스킬의 테스트 결과 수집
- 리포트 생성 및 저장

## 결과 저장 경로
```
/home/peterchung/HWTestAgent/test-results/MyTester/
├── reports/           # 마크다운 리포트
├── screenshots/       # E2E 스크린샷
└── logs/              # API 로그 (통합)
```

## 파일명 규칙
- 리포트: `YYYY-MM-DD-[프로젝트명]-[단위|통합|E2E|스모크]-테스트.md`
- 스크린샷 폴더: `YYYY-MM-DD-[프로젝트명]-screenshots/` (E2E만)

## 사용 예시
```
/스킬테스터                              # 현재 프로젝트 단위 테스트
/스킬테스터 세일즈허브 단위 테스트        # → 스킬테스터-단위 호출
/스킬테스터 허브매니저->세일즈허브 통합    # → 스킬테스터-통합 호출
/스킬테스터 오라클에서 허브매니저->세일즈허브 E2E  # → 스킬테스터-E2E 호출
/스킬테스터 허브매니저 스모크            # → 스킬테스터-스모크 호출
/스킬테스터 핀허브 smoke test           # → 스킬테스터-스모크 호출
```

## 공통 결과 표시 규칙
- ✅ 통과 (Pass)
- ❌ 실패 (Fail)
- ⚠️ 경고 (Warning)
- 🔧 수정됨 (Fixed)
- 통과율: N/M (XX%)
